# Memory Management - Documentation

Last Updated: 2025-12-05
Maintainers: Community

---

## BEGINNER

### C Language Specification (ISO/IEC)
Maintainer: International Standards Organization
Type: Language Standard
Link: https://en.wikipedia.org/wiki/C_(programming_language)#Standards
Version: C11/C17
Description: Official C language standard covering memory model, pointers, and allocation semantics.
Why included: Authoritative specification; defines memory management behavior at language level.

### Rust Book - Memory Safety (Official)
Maintainer: Rust Project
Type: Official Language Guide
Link: https://doc.rust-lang.org/book/ch04-00-understanding-ownership.html
Description: Official Rust documentation on ownership, borrowing, and memory safety without garbage collection.
Why included: Modern language design philosophy; shows alternative to traditional memory management.

---

## INTERMEDIATE

### Linux Memory Management (Kernel Documentation)
Maintainer: Linux Kernel Project
Type: System Documentation
Link: https://www.kernel.org/doc/html/latest/admin-guide/mm/
Description: Comprehensive kernel documentation on Linux memory management, paging, and virtual memory.
Why included: Operating system perspective; shows how theory manifests in real systems.

### Java Memory Model (Official)
Maintainer: Java Platform Group (Oracle)
Type: Official Specification
Link: https://docs.oracle.com/javase/specs/jvms/se17/html/jvms-2.html#jvms-2.5
Version: Java 17+
Description: Official Java Virtual Machine specification including memory model and garbage collection contract.
Why included: Production language reference; defines memory semantics for JVM languages.

## ADVANCED

### POSIX Memory Management (Official Standard)
Maintainer: IEEE/The Open Group
Type: Standards Specification
Link: https://pubs.opengroup.org/onlinepubs/9699919799/
Description: POSIX standard covering memory allocation, protection, and management APIs.
Why included: Systems programming standard; defines portable memory management interfaces.

### Intel x86-64 Memory Architecture (Technical Reference)
Maintainer: Intel
Type: Processor Architecture Documentation
Link: https://www.intel.com/content/www/us/en/architecture-and-technology/64-bit-isa-manual-suite.html
Description: Official Intel documentation on x86-64 memory hierarchy, paging, and protection.
Why included: Hardware perspective; essential for understanding memory performance and security.

## RECENT (2023+)

### Memory Tagging Extension (ARM) Documentation
Maintainer: ARM Holdings
Type: Architecture Extension Specification
Link: https://developer.arm.com/documentation/
Version: ARMv8.5+
Description: Documentation on hardware-based memory safety through MTE (Memory Tagging Extension).
Why included: Emerging hardware support for memory safety; future direction of architecture design.

## BEGINNER

### Automatic Heap Sizing: Taking Real Measure of Garbage Collection
Author: David Detlefs, Christine Flood, Steve Heller, Tony Printezis
Year: 2002
Venue: ACM SIGPLAN Notices (OOPSLA)
DOI: 10.1145/584701.584735
Link: https://dl.acm.org/doi/10.1145/584701.584735
Description: Practical analysis of heap sizing strategies in garbage-collected systems with clear explanations of GC dynamics.
Why included: Accessible paper explaining how GC decisions impact performance; bridges theory and practice.

### The Memory Autobiography: Applying Program Analysis to Understand Memory Use
Author: Brian G. Ryder, Frank Tip
Year: 2007
Venue: ACM Transactions on Software Engineering and Methodology
Description: Practical techniques for understanding and analyzing memory behavior in programs.
Why included: Tools and methods for debugging memory management issues; practical orientation.

## INTERMEDIATE

### Garbage Collection: Algorithms for Automatic Dynamic Memory Management (Survey)
Author: Richard Jones
Year: 1996
Venue: ACM Computing Surveys
DOI: 10.1145/242223.242236
Link: https://dl.acm.org/doi/10.1145/242223.242236
Description: Comprehensive survey of garbage collection algorithms including mark-and-sweep, copying, generational, and incremental collection.
Why included: Definitive survey paper; synthesizes research on GC into unified framework.

## ADVANCED

### Efficient and Robust Garbage Collection Strategies
Author: Jennifer G. Rexford, Claude W. Anderson, Rajiv Gupta, Scott F. Kaplan
Year: 1998
Venue: ACM Transactions on Programming Languages and Systems
DOI: 10.1145/291891.291893
Link: https://dl.acm.org/doi/10.1145/291891.291893
Description: Theoretical analysis and practical strategies for concurrent garbage collection with proofs of correctness.
Why included: Rigorous treatment of concurrent GC challenges; research-level contribution.

### Dynamic Memory Allocation and Fragmentation in the Linux Kernel
Author: Mel Gorman
Year: 2004
Description: Deep technical analysis of kernel-level memory management and page reclamation strategies.
Why included: Systems-level perspective on memory management challenges and solutions.
